# Review

When reviewing:

1. Prioritize concrete bugs, behavioral regressions, security issues, and
   missing tests.
2. Cite specific files, symbols, or commands when possible.
3. Separate findings from suggestions.
4. Keep summaries brief unless the user asks for more detail.
5. If no issues are found, say so clearly and note any residual risk.
